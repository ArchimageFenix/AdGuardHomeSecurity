# Público objetivo y alcance del proyecto

## Público objetivo

Esta guía está orientada principalmente a:

- Usuarios de VPS personales
- Homelabs
- Ingenieros independientes
- Administradores de servidores pequeños
- Personas que exponen servicios críticos sin infraestructura empresarial
- Usuarios que no cuentan con WAF comerciales, SIEM ni SOC dedicados
- Entornos con recursos limitados (poca RAM, pocos cores, presupuesto reducido)

Está especialmente pensada para quienes desean:

- Exponer servicios como DNS, paneles web u otros de forma segura
- Minimizar superficie de ataque
- Mantener control total de su infraestructura
- Aprender desde la experiencia real, no desde configuraciones ideales de laboratorio

---

## Qué sí busca este proyecto

- Mostrar un enfoque práctico de hardening en entornos reales
- Documentar decisiones técnicas basadas en observación directa
- Enseñar cómo proteger servicios sin depender de software empresarial
- Compartir lecciones aprendidas tras errores, incidentes y ajustes
- Priorizar eficiencia, simplicidad y seguridad por capas

---

## Qué NO pretende ser este proyecto

- No pretende ser una solución empresarial
- No reemplaza un WAF profesional
- No sustituye un SIEM, un SOC o soluciones XDR
- No es una guía universal para todos los escenarios
- No promete seguridad absoluta

Este proyecto asume una realidad clara:

> La seguridad perfecta no existe. Solo existen decisiones cada vez más difíciles de atacar.

---

## Alcance técnico

El enfoque principal se basa en:

- AdGuard Home
- DNS cifrado (DoH / DoT)
- Fail2ban
- ipset
- journald / journalctl
- Tailscale
- Unbound
- Firewall a nivel de sistema
- Automatización ligera mediante scripts

Todo el contenido está limitado estrictamente a herramientas disponibles en un entorno Linux estándar.
